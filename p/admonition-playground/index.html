    Admonition Playground

    This page demonstrates colored admonition blocks using pure Markdown fenced blocks.

    You can mix:

    • inline math like \(\hat{\epsilon}_S(h)\)
    • display math
    • lists and nested lists
    • normal code blocks
    • links and references

    1. Definition + inline math

    Definition (Empirical Risk)

    Given a hypothesis \(h\), a dataset \(S=\{(x^{(i)},y^{(i)})\}_{i=1}^m\), define the empirical error:

    \[ \hat{\epsilon}_S(h) \;=\; \frac{1}{m}\sum_{i=1}^m \mathbf{1}\{h(x^{(i)}) \neq y^{(i)}\}. \]

    We will also write \(\hat{\epsilon}(h)\) when \(S\) is clear from context.

    2. Theorem + proof (with steps)

    Concentration of Bernoulli Mean

    Let \(Z_1,\dots,Z_m\) be i.i.d. Bernoulli random variables with mean \(\phi\), and let

    \[ \hat{\phi}=\frac{1}{m}\sum_{i=1}^m Z_i. \]

    Then for any \(\gamma>0\), \(\hat{\phi}\) concentrates around \(\phi\) as \(m\) grows.

    Proof Sketch
    1. View \(\hat{\phi}\) as the average of bounded i.i.d. random variables.
    2. Apply a concentration inequality (e.g. Hoeffding).
    3. Rearrange the bound into the form “probability of deviation ≤ something”.

    This is a sketch; details depend on the exact inequality you choose.

    3. Proposition + Corollary

    Overfitting Indicator
    If a model has very low training error but high validation error, it likely has high variance (overfitting).
    Model Selection
    Cross-validation is a practical tool to estimate generalization error and select model complexity.

    4. Example + list + nested list

    k-fold Cross Validation

    To run \(k\)-fold cross validation:

    • Split the dataset into \(k\) folds: \(S_1,\dots,S_k\).
    • For each fold \(j\):
      • Train on \(S \setminus S_j\)
      • Evaluate on \(S_j\)
    • Average the \(k\) validation errors.

    A typical choice is \(k=10\).

    7. Warning / Caution / Danger

    Math Rendering
    If one LaTeX expression is malformed, KaTeX may skip rendering that expression. (We configured it to avoid breaking the whole page.)
    Markdown Pitfall
    Avoid mixing $...$ math with code spans like `$x$`.
    KaTeX ignores code blocks by default.
    Security
    Do not publish secrets (tokens/keys) in Markdown posts.
